The repository includes a machine-verifiable conformance surface so that compatibility claims can be grounded in observable behavior rather than prose alone.

## What to evaluate

A baseline-compatible implementation should make it possible to inspect:

- query and lookup behavior,
- schema validation expectations,
- revocation and freshness semantics,
- the governance artifacts relied upon,
- and the evidence emitted from concrete runs.
