# Roadmap

This roadmap is intentionally practical. It focuses on the moves that most improve implementability and adoption.

## Immediate priorities

### 1. Tighten discovery conventions
The protocol becomes easier to integrate when developers know exactly how to discover namespaces, directories, and schema usage.

### 2. Add conformance and validation guidance
A lightweight conformance profile would help publishers and consumers verify that their implementations are aligned.

### 3. Expand examples into end-to-end flows
Developers adopt faster when examples show not just payloads, but actual verification and query flows.

### 4. Clarify versioning policy
Schema evolution, compatibility expectations, and deprecation handling should be explicit.

### 5. Improve contribution process
A lightweight protocol-change proposal process would make evolution easier to track and discuss.

## Medium-term priorities

- More schema families for common trust use cases.
- Example client and server implementations.
- Better federation and cross-registry discovery guidance.
- Ecosystem mappings beyond current included schemas.

## Strategic goal

The long-term goal should be simple: make DeDi feel less like an interesting concept and more like a protocol developers can confidently build into real systems.
