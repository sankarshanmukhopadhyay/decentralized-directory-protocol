# Security policy

## Reporting a vulnerability

Please report suspected vulnerabilities privately to the project maintainers before public disclosure.

A high-quality report should include:

- affected component,
- impact,
- reproduction steps,
- proof-of-concept if safe to share,
- and suggested mitigations if available.

## Coordinated disclosure

The project aims to acknowledge reports promptly, investigate impact, and coordinate disclosure once fixes or mitigations are available.

## Scope

This repository contains protocol and schema artifacts. Some vulnerabilities may be repository-level design issues, while others may only emerge in specific deployments.
